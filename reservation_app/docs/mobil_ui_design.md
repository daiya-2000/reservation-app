# モバイルUIデザイン

このドキュメントは、居住者向けモバイル画面のUI方針と実装反映内容を整理したものです。  
元のHTMLモックをそのまま保持するのではなく、Flutter実装に落とし込んだ時点の判断基準と画面ごとの要点をまとめています。

## 共通方針

- 全体背景は明るいグレー `#F7F9FB`
- メインカラーは青緑系 `#004D64` / `#0D5D78`
- 大きな角丸カードと淡いグラデーションをベースにする
- 情報は 1 画面 1 主目的になるよう整理する
- 既存の CRUD / Firestore フローは維持し、見た目と操作性を改善する
- 下部ナビゲーションは既存構成をそのまま使う

## 対象画面

### 施設予約画面

- ヒーローカードでマンション名を表示
- 施設一覧は画像付きカードで表示
- 料金や説明はチップと補助文で整理
- タップ時の導線は施設詳細画面へ遷移
- 既存の施設取得処理を維持

実装:
- `lib/reservation_tab.dart`

### 施設詳細画面

- 画像ヘッダー + 情報カードの構成
- 予約カレンダーは 7 日表示
- 縦スクロール時に時間列と予約コマがずれない
- 横スクロール時に日付ヘッダーと予約コマが同期する
- 終了時刻は非包含で扱い、`15:00-17:00` の次に `17:00-18:00` を予約可能
- `24:00` 終了を選択可能

実装:
- `lib/facility_detail_screen.dart`

### 掲示板画面

- 上部ヒーローで画面の役割を明示
- 投稿一覧はカード化
- 投稿日・PDF有無を視覚的に判別しやすくする
- 投稿詳細は下部シートで確認
- PDF表示導線は既存仕様を維持

実装:
- `lib/bulletin_tab.dart`

### マイページ

- プロフィールカードを主軸に再構成
- マンション名、部屋番号、氏名、メールを整理表示
- 機能導線は用途別にまとめる
- ログアウトは右上の独立アクションへ配置

実装:
- `lib/home_tab.dart`

### お問い合わせ画面

- フォームと回答履歴をモダンなカードUIへ変更
- タブは見やすいピル型デザインを採用
- 回答履歴は状態やカテゴリをチップで整理
- 返信導線はシート形式で扱う

実装:
- `lib/contact_tab.dart`

### 通知画面

- ヒーローカード + 未読サマリーを表示
- 通知一覧は既読 / 未読の差が分かるカードデザイン
- タップで既読化する既存動作を維持

実装:
- `lib/notification_tab.dart`

### ログイン画面

- ヒーローカード + 入力カードの二段構成
- パスワードの表示切替を追加
- ログイン中はボタンをローディング表示
- モバイルでは `居住者アカウントでアクセス`
- Web では `管理者アカウントでアクセス`

実装:
- `lib/login_screen.dart`

## 実装時の注意

- デザイン変更で Firestore のスキーマや既存ルート名は変えない
- 既存のユーザー権限判定は維持する
- 予約、掲示板、お問い合わせ、通知の既存データはそのまま扱う
- `hot reload` で反映できない変更が入る場合は `hot restart` を使う
